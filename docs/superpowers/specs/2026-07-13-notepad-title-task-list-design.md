# Notepad 动态标题与任务列表样式设计

## 目标

- 顶栏始终显示当前文章名称，切换、创建和重命名时立即更新。
- 保留 Vditor 已有的 Markdown 任务列表行为，只优化其视觉样式。
- 不改写 `hybrid-editor.js` 的回车、光标、标记渲染或保存逻辑。

## 动态标题

- 使用 `currentNotepads` 中当前文章的 `name` 作为唯一标题来源。
- 文章选中后立即更新顶栏，不等待正文网络请求完成。
- 缓存恢复、新建文章、重命名成功和重命名回滚均同步顶栏。
- 阅读模式与编辑模式使用相同的文章标题，不再在编辑模式恢复站点名。
- 浏览器标签继续使用“文章名 - DumbPad”，无有效文章时回退到站点名。
- 标题过长时沿用现有单行省略和原生 `title` 提示。

## 任务列表

- Markdown 语法保持不变：`- [ ]` 表示未完成，`- [x]` 表示完成。
- 回车续建、空项退出、勾选写回 Markdown、撤销和保存全部交给 Vditor 原生实现。
- 不新增任务列表事件监听器，不阻止或代理 Vditor 的点击与键盘事件。
- 复选框使用 17px 圆角方框；未完成态为低对比度灰色边框，完成态使用现有金色主色。
- 完成项文字降低对比度并使用细删除线；未完成文字保持正文对比度。
- 提供轻量悬停、按下和 `:focus-visible` 状态，不使用阴影或独立卡片背景。
- 浅色、深色主题共用现有颜色 token；移动端保持足够的点击区域和正文节奏。

## 保护范围

- 不修改 `hybrid-editor.js` 的任务列表行为。
- 不修改 Markdown 存储格式、同步协议、撤销栈或 Vditor 配置。
- 不影响普通无序列表、有序列表、Thought 子任务或预览导出。

## 验证

- 切换两篇文章时，顶栏和浏览器标签同步变化。
- 创建、重命名及重命名失败回滚时，顶栏名称正确。
- 输入两条任务列表，勾选与取消后 Markdown 在 `[ ]` 和 `[x]` 间变化。
- 在非空任务末尾按回车创建下一条未完成任务；空任务按回车退出列表。
- 检查浅色、深色主题以及 390px 移动端布局。
- 运行项目静态检查、相关回归测试，并在最终交付前执行完整本地测试集。
