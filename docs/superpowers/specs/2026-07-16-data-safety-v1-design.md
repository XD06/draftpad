# DumbPad 数据安全 V1 设计

## 状态与决策

本文记录已经确认的 V1 方向：DumbPad 是单一管理员个人应用，数据安全优先于功能扩展。本文是设计说明，不包含凭证、桶名真实值或任何迁移命令。

已确认：

- 一个管理员账号，长主密码 + TOTP；日常可信设备使用方便，但高危操作必须再次验证。
- 正常 AI 功能保持运行；当前自动 Thought 处理、手动 Insight 与只读 Agent 不因本项目而关闭。
- 运行数据、备份数据与本地备份必须分离；七牛不进入自动备份链路。
- 空间级删除必须先归档并可恢复，不能由普通 UI 或普通 API 直接永久删除。
- 备份容量有限。每个独立备份仓库上限为 `1 GiB`，不降低原图或附件质量。
- 恢复只能写入新的数据空间；不得覆盖正在运行的数据空间。

## 1. 现状与风险

当前项目已具备文章/Thought 的条目级垃圾桶、`version/baseVersion` 乐观并发、前端启动缓存和只读 Agent 工具白名单，但仍有以下空间级风险：

1. `DUMBPAD_PIN` 同时被当作网页登录 Cookie 和完整 Bearer API 凭证，缺少会话、设备和权限范围。
2. `routes/data-management-routes.js` 可执行 S3 prefix 删除与“本地覆盖 S3”；精确 prefix 二次输入不是独立权限边界。
3. `scripts/s3-prefix-tools.js` 和真实 S3 测试脚本具有删除能力。
4. 当前备份没有定期快照、独立密钥、容量约束、恢复演练或不可与主数据一起误删的隔离面。
5. 当前 R2 桶含有不属于 DumbPad 的其他对象。长期 S3 凭证的主要隔离单元是桶，而不是普通 key prefix；因此 prefix 不可作为生产隔离的唯一防线。

本设计不宣称能抵御 VPS root 被完全控制的攻击。V1 的目标是大幅降低应用缺陷、误操作、错误同步、泄露普通会话和错误脚本造成的删除范围，并保证可验证恢复。

## 2. 范围与非目标

### V1 范围

- 独立运行桶与备份桶，迁移和恢复均采用新空间验证后切换。
- 加密、去重、受容量限制的本地/R2 备份仓库。
- 空间级归档和恢复流程，禁用直接永久删除。
- 单管理员密码、TOTP、恢复码、可信设备、会话与 API token 范围。
- 高危操作提升验证、审计、同步防覆盖和恢复演练。
- 登录界面随认证流程一并重做，视觉优化独立于安全核心。

### 非目标

- 多用户、团队角色、组织 SSO 或细粒度分享权限。
- CRDT/OT 多端协同编辑。
- 让 Agent 自动修改内容、管理数据空间或调用对象存储。
- 把 AI 模型调用改为默认关闭；既有 AI 工作流按原行为保留。
- 将备份系统设计成针对服务器完全失陷的不可变合规存档。

## 3. 三平面边界

```text
浏览器 / API token
        │
        ▼
认证与权限平面 ── 高危操作提升验证 ── 操作审计
        │
        ▼
内容平面：DumbPad 服务 ──────────────── R2 运行桶 dumbpad-prod
        │                                        │
        └── 受控只读导出 ── 宿主机备份服务 ──────┴── R2 备份桶 dumbpad-backup
                                  │
                                  └── /var/lib/dumbpad-backups
```

### 内容平面

- 应用容器只拥有 `dumbpad-prod` 的对象读写凭证，且该桶只存 DumbPad 运行数据。
- 正常 Notepad、Thought、附件、图片、同步和现有 AI 队列都留在内容平面。
- 前端永不持有 S3 凭证，仍只调用后端 API。

### 备份平面

- 独立的宿主机 `systemd` 服务/定时器执行备份，不作为应用容器启动命令的一部分。
- 备份服务持有：运行桶的只读凭证、备份桶的读写凭证、备份主密钥；这些凭证位于 root 只读的独立配置文件，不进入项目 `.env`，不挂载进应用容器。
- 应用容器没有备份桶凭证，因而应用 UI、普通 API、Agent 和应用中的删除缺陷不能删除备份。

### 认证与控制平面

- 持久认证状态位于宿主机独立路径，例如 `/var/lib/dumbpad-security`，不与用户正文或 S3 主数据共用。
- 保存密码哈希、TOTP 加密密钥、恢复码哈希、会话/可信设备 token 哈希、API token 哈希、提升验证记录和审计链。
- 文件权限为服务账号可读写、其他账号不可读；不得加入 Git、普通备份明文日志或前端响应。

## 4. 运行与备份桶

### 桶隔离

- 新建专用运行桶 `dumbpad-prod`，应用 R2 token 仅授予该桶 Object Read & Write。
- 新建专用备份桶 `dumbpad-backup`，其凭证不提供给应用容器。
- 旧共享桶不在迁移后删除；先保留至少 30 天并完成恢复演练，再由人工决定后续处置。
- 备份桶不是普通运行数据空间，不能通过“切换数据空间”选择，也不能由设置页面浏览、删除或改写。

### 迁移原则

1. 先建立并验证备份链路，再复制旧数据到新的运行桶/新 prefix。
2. 对象数、总字节、关键清单 hash 与可读取性均通过后才允许切换。
3. 切换仅修改部署配置并重建容器；不原地覆盖旧空间。
4. 出现异常时切回旧空间或恢复到另一个新空间，不执行清空式“回滚”。

## 5. 备份格式、去重与容量

### 5.1 可恢复数据与可重建数据

备份的 canonical 集合包含：Notepad 清单与正文、Thought 本体、用户标签/子任务、手动 relation、suppressed relation、垃圾桶 payload、文章原始附件和图片原图。

不进入备份的派生集合包含：图片 preview/thumbnail、搜索索引、前端缓存、AI meta、AI relation、Agent 运行中缓存。这些内容恢复后可重建；用户手工 relation 与用户正文绝不视为派生数据。

### 5.2 去重加密仓库

每次快照不复制一整套文件，而是生成一个不可变快照清单：

1. 对 canonical 文件计算稳定的内容标识 `HMAC-SHA256(HKDF(BACKUP_MASTER_KEY, "dedup"), 原始字节)`，避免直接暴露明文 hash，也不额外引入一份独立 dedup 密钥。
2. 仅当备份仓库中不存在该标识时，先压缩再使用 AES-256-GCM 加密为内容块；记录随机 nonce、认证 tag、原始长度与压缩算法。
3. 快照清单保存“逻辑路径 → 内容标识、长度、元数据”的映射，并单独加密、签名和记录完整性 hash。
4. 后续快照复用未变更内容块；同一张原图、附件或正文不会随着保留代数重复占用空间。

文本和 JSON 通常压缩收益显著。JPEG、PNG、视频、PDF、ZIP 等已压缩文件会跳过无收益压缩，但始终保存原始字节，不能为节省空间降低图片质量。

本地仓库和 R2 备份桶采用相同清单/块格式，但分别独立保存。它们是两份完整恢复副本；每份都限制为 `1 GiB`。这是两地可恢复的必要冗余，不等同于把同一内容按快照次数复制十几份。

### 5.3 保留、回收与配额

- 保留 3 个每日、4 个每周、6 个每月恢复点。
- 高危操作前快照最多 3 个，保留 14 天。
- 快照到期先移动到备份垃圾桶，保留 14 天；只有不被任何活动快照或垃圾桶清单引用的内容块才可回收。
- `BACKUP_MAX_BYTES=1073741824` 是每个备份仓库的硬上限；70% 产生预警，90% 产生强预警。
- 达到上限时不删除活动数据、不删除未过宽限期备份；暂停新的自动快照并阻止高危操作。附件上传在预估会使可恢复集合超限时被拒绝，纯文本编辑仍可保存但界面显示“备份降级”。
- 容量和保留数量通过配置声明，不写死在业务代码；所有自动回收写审计事件。

### 5.4 一致性与验证

备份服务创建快照前读取主清单、所引用对象和最终主清单版本；若版本变化则重试，避免得到清单与正文不一致的集合。高危操作必须等待一次通过验证的快照。

每次备份输出不含正文的报告：快照 ID、对象数、原始/压缩/去重后字节、清单 hash、容量占用、错误和耗时。恢复演练必须验证清单 hash、对象数、随机抽样原文 hash 和图片原始字节 hash。

## 6. 空间级归档、删除与恢复

### 删除改为归档

- 生产环境默认禁用 `POST /api/data-management/s3/delete` 和 `local-overwrite-s3` 的执行路径；只允许盘点、只读预览和备份状态查询。
- 用户发起“归档数据空间”时，先完成提升验证、精确输入空间 ID、创建并验证高危快照，再将该空间标记为 archived。
- 归档空间从普通选择列表隐藏，但源对象不立刻删除；恢复从备份写入新的 prefix，再由用户确认切换。
- V1 不提供 UI 永久删除空间。任何未来硬删除只能通过离线运维命令执行，至少等待 30 天，并再次验证备份和空间 ID。

### 恢复流程

1. 在备份列表选择快照并进行提升验证。
2. 恢复服务创建全新的目标 prefix，拒绝非空目标。
3. 解密、解压并重建所有 canonical 文件；不复制派生数据。
4. 校验 manifest、对象数、字节数和抽样原始 hash。
5. 用户查看恢复报告后，显式把该 prefix 设为活动空间并重启/重载服务。

当前活动空间从不作为恢复写入目标。

恢复不是备份服务的反向写入。它是单独的一次性宿主机任务：只在提升验证后，为一个明确且为空的目标 prefix 申请短时写入能力；备份服务本身始终只能读取运行桶，不能向运行桶写入或删除。

## 7. 认证、可信设备与 API token

### 管理员认证

- 首次迁移时，现有 PIN 仅用于进入一次性安全设置流程：设置长主密码、绑定 TOTP、下载 10 个一次性恢复码。
- 密码使用经参数化的现代 KDF 保存哈希；TOTP 密钥加密后保存；恢复码仅保存哈希。
- 迁移完成后废弃 PIN Cookie 和 PIN Bearer 登录方式。浏览器 Cookie 改为随机、不透明、HttpOnly、Secure、SameSite=Strict 的会话 ID。
- 正常会话可持续使用；可信设备有效期 60 天，最多 5 台。可信设备允许“主密码后跳过 TOTP”，不能跳过高危操作的 TOTP。
- 提升验证有效期最多 10 分钟，只用于数据空间归档/恢复/切换、完整导出、凭证管理和 token 管理。
- 设备页面展示名称、最近使用时间和大致来源，支持立即撤销其他设备；不采用隐蔽浏览器指纹。

### API token

- API token 是独立随机 token，仅显示一次，服务端只保存哈希。
- token 必须可命名、过期、撤销，并带 scopes：`content:read`、`content:write`、`thoughts:read`、`thoughts:write` 等。
- 不存在能够通过普通 API token 获得的 `storage:admin`、`backup:admin` 或 `auth:admin` scope。
- 旧 PIN Bearer 文档和 OpenAPI 在迁移期明确标为废弃，完成迁移后移除。

## 8. 同步、AI 与审计

### 同步

- 保留当前 `baseVersion`/409 冲突保护和本地 dirty 缓存；缓存不得自动覆盖服务端较新内容。
- 禁止自动全量 local → S3 覆盖；任何迁移都是显式、可盘点、可验证、写入新空间的流程。
- 日常保存和同步失败不得阻塞写作；显示清晰的同步降级/冲突状态。只有高危动作因备份不健康而被阻止。

### AI

- 保留现有 Thought 自动 AI 队列、Insight 和已实现的只读 Agent 行为。
- 现有 Agent 继续由服务端工具白名单、候选集、上下文和步数预算约束；模型没有 S3 凭证、数据管理路由、认证状态或备份密钥。
- 未来如增加 AI 写入能力，必须返回用户可见 diff，并在用户确认后走普通的带 `baseVersion` 内容 API；不得让模型直接执行删除、空间切换或备份操作。

### 审计

记录高危认证、设备信任/撤销、token 创建/撤销、空间归档/恢复/切换、备份/回收、同步冲突和被拒绝高危请求。日志只保存时间、设备/会话标识、IP、动作、对象数、结果和关联快照 ID；不保存正文、密码、token、TOTP 或对象存储密钥。

审计日志采用追加和哈希链，并由宿主机备份服务定期复制到备份桶。V1 的审计用于误操作追溯和告警准备，不应被宣传为抵御 VPS root 攻击的不可篡改证据。根据当前决定，V1 仅记录日志，不发送即时通知。

## 9. 实施顺序

1. **止血开关**：生产默认禁用空间硬删除与本地覆盖云端；为危险 CLI 增加生产环境拒绝、精确目标与显式一次性确认。
2. **备份核心**：实现独立块仓库、清单、加密、容量/回收、备份报告和 restore-to-new-prefix；先在 local/mock S3 验证。
3. **部署隔离与迁移**：用户创建两个 R2 桶和最小权限 token；备份凭证只写入宿主机 root 配置；复制旧数据、恢复演练、再切换运行桶。
4. **控制平面认证**：实现密码/TOTP/恢复码、会话、可信设备、提升验证与 scoped API token；提供 PIN 一次性迁移路径。
5. **管理界面**：将数据管理改为只读状态、备份/恢复报告和归档入口；新增设备管理；重做登录/二次验证页面。
6. **强化验证**：接入审计界面、容量提示、定时恢复演练说明，最后清理遗留的 PIN 文档和危险 UI。

每步都应独立提交、可回退，并在下一步前完成测试。不得混入编辑器、Thought 交互或 AI 工作流的无关重构。

## 10. 验证清单

- 应用 token 无法访问备份桶；备份服务无法向运行桶写入或删除。
- 运行桶内删除、错误同步、应用异常后，能从本地和备份桶各自恢复到新 prefix。
- 相同附件跨多个快照只存一个内容块；图片/附件恢复后的字节 hash 与原文件一致。
- 达到 70%、90%、100% 容量阈值时，提示、阻止策略和审计符合本文。
- 归档不会删除源对象；普通会话、API token、Agent 和 UI 均不能调用空间硬删除。
- PIN 迁移完成后，PIN Cookie/Bearer 失效；失效设备、过期会话、撤销 token 和无 TOTP 的高危操作均被拒绝。
- TOTP、恢复码、可信设备和高危提升验证都具备正反向测试。
- AI 正常运行，但模型工具和外部请求中不出现备份密钥、S3 密钥、认证 secret 或管理能力。
- 每月至少完成一次隔离恢复演练；演练失败即视为备份不健康，并阻止高危操作。

## 11. 部署前人工输入

以下仅在实际部署时由管理员提供，文档和应用日志不得记录其值：运行桶/备份桶名称、两套最小权限 R2 凭证、备份主密钥、认证主密钥、主密码、TOTP 初始化信息和 VPS 实际备份挂载路径。默认本地路径为 `/var/lib/dumbpad-backups`，默认调度时区为 `Asia/Shanghai`。
