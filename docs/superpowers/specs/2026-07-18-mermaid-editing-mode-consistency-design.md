# Mermaid 编辑模式一致性设计

## 目标

统一 Mermaid 在文章编辑器中的表现，消除“粘贴时短暂显示图表、刷新后显示源码”的不一致，同时避免复杂或错误 Mermaid 阻塞所见即所得编辑器。

## 行为约束

- 所见即所得编辑模式始终把 Mermaid 作为代码块显示，并保留语言工具栏。
- 阅读模式才把 Mermaid 渲染为图表。
- 源码模式始终使用标准的 `mermaid` 围栏语言，不暴露内部显示语言。
- 粘贴包含 Mermaid 围栏的纯文本时，在交给 Vditor 前转换为内部显示语言，阻止原生 Mermaid 渲染抢先执行。
- 在源码文本框或已有代码块内部粘贴时保持普通文本粘贴行为，不改写用户内容。

## 数据流

1. 文章加载、切换或粘贴 Markdown 时，将围栏语言 `mermaid` 转换为内部的 `dumbpad-mermaid`。
2. 编辑器按普通代码块展示内部语言，代码工具栏仍向用户显示 `mermaid`。
3. 读取、保存或切换源码模式时，将内部语言恢复为标准 `mermaid`。
4. 进入阅读模式时，根据代码块的标准语言创建独立图表层，不修改原始代码内容。

## 容错

- 剪贴板没有纯文本、没有 Mermaid 围栏或编辑器不支持 Markdown 插入时，退回 Vditor 默认粘贴。
- Mermaid 解析失败只影响阅读态图表，编辑态源码仍可修改和保存。

## 验证

- 粘贴 Mermaid 后，编辑模式不出现图表闪烁，只显示 Mermaid 代码块。
- 保存并刷新后，源码仍包含 `mermaid`，编辑模式仍显示代码块。
- 阅读模式正常渲染图表。
- 普通文本、普通代码块、源码模式粘贴和图片粘贴行为不变。
