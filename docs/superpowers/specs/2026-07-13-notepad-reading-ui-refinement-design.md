# Notepad 阅读界面精修设计

## 目标

在不改变 DumbPad 现有视觉语言和交互结构的前提下，提升桌面端 Notepad 的阅读舒适度与整体协调感。保留悬浮圆角顶栏、左右侧栏、金色图标和浮动工具；移动端只做必要的间距与拥挤度修正。

## 范围

### 桌面端

- 顶栏继续悬浮在页面上方，改为可感知的透明磨砂玻璃材质。
- 左侧目录、中央正文、右侧最近编辑使用同一套半透明表面 token。
- 三个区域通过统一圆角、边缘高光和间距形成整体，不再依赖突兀的硬分隔线。
- 正文限制在约 680–720px 的阅读宽度，优化字号、行距、标题间距和列表节奏。
- 保留复制、编辑、滚动辅助等现有功能和位置语义。

### 移动端

- 保持现有导航结构和整体外观。
- 仅调整顶部按钮间距、正文左右留白和重复工具的拥挤度。
- 不引入桌面端三栏玻璃面板布局。

### 不在范围内

- 不改变 Notepad、Thought、搜索、设置或同步的数据逻辑。
- 不删除、合并或重命名现有功能入口。
- 不重构编辑器 DOM 或替换现有编辑器。
- 不改变 Thought 页面已确认的样式。

## 视觉规范

### 顶部栏

- 保持当前悬浮圆角轮廓和左右操作分布。
- 推荐表面色：`rgba(249, 249, 252, 0.42)`。
- 推荐背景模糊：`blur(22px)`；饱和度：`saturate(1.22)`。
- 使用轻量白色边缘高光与不超过 8px 模糊的阴影。
- 正文滚动容器设置顶部安全区和 `scroll-padding-top`，避免锚点或当前阅读行停在顶栏正下方。
- 为不支持 `backdrop-filter` 或偏好减少透明度的环境提供较实的表面回退。

### 三个内容区域

- 左侧栏、正文区、右侧栏共用同一组背景、边框、圆角和模糊变量。
- 桌面端区域间距约 10px，取消高对比度竖向分割线。
- 面板表面建议使用约 30% 白色叠加和 10px 模糊。
- 左侧选中项使用低饱和度金色底和小型状态点，不再使用整行高饱和度金色。
- 右侧最近编辑使用紧凑文本列表；空状态保持与列表一致的密度。

### 正文

- 正文最大宽度 720px，常规目标宽度约 680px。
- 桌面端正文建议 15–16px，行距约 1.75–1.85。
- 标题与正文使用现有字体体系，不新增远程字体依赖。
- 标题、段落、列表、代码块、表格和附件需要在同一阅读宽度下检查。
- 正文不额外套纸张卡片，不引入横线或明显纸纹。

### 动效与状态

- 仅对背景色、边缘色、透明度和 transform 使用 150–220ms 的状态过渡。
- 按钮按下提供轻微缩放反馈，触摸设备不依赖 hover。
- `prefers-reduced-motion` 下移除位移动效；`prefers-reduced-transparency` 下提高背景不透明度并取消模糊。

## 功能保护

- 顶栏所有按钮的 DOM、事件绑定、快捷键和可访问名称保持不变。
- 左右侧栏现有展开、选择、最近编辑和响应式行为保持不变。
- 编辑模式、阅读模式、打印、下载、复制、搜索和同步状态均需回归验证。
- 仅在现有 CSS 和必要的样式状态类中实现，不迁移业务逻辑。

## 验证

- PC：1280×720、1440×900，检查空状态、短文、长文和滚动状态。
- Mobile：390×844，确认顶部按钮、正文留白、侧栏抽屉和浮动工具无回归。
- 检查浅色与深色主题、阅读与编辑模式、`prefers-reduced-motion` 和无 `backdrop-filter` 回退。
- 浏览器截图对比顶部通透度、三栏协调度、正文行宽和滚动遮挡。
- 执行项目完整本地测试；真实 S3 测试仅在具备凭据时运行。
