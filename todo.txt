# 3. iOS 视觉设计系统

## 3.1 设计原则

- **不要改变** 现有 HTML 结构、DOM 层级、JS 交互逻辑
- **不修改** 任何功能行为和交互手势（click / dblclick / longpress 保持不变）
- **保留现有 class 命名**，只覆盖 CSS 视觉属性
- **目标风格**：iOS Native UI / Apple Human Interface / iOS Settings 风格
- **禁止**：glassmorphism 炫技、Dribbble 风、重阴影、大渐变、设计感过强

# 4. 实施计划

## Phase 1：数据重构 + AI 后端（server 层）

```
1. 创建目录结构：
   data/thoughts/
   data/thoughts.meta/
   data/relations/

2. 修改 server.js：
   - POST /api/thoughts 改为存储到 thoughts/{id}.json
   - 新增异步 AI 处理队列
   - PATCH/DELETE 对应更新
   - 新增 GET /api/thoughts/:id/relations

3. 实现 AIProvider 层
   - OpenAICompatibleProvider
   - 关键词提取 + embedding 生成

4. 回填脚本：遍历已有 thoughts，生成 .meta.json + relations
```

## Phase 2：前端关联 UI

```

6. 关联面板
   - 展示关联 Thoughts 列表（卡片形式）
   - 点击跳转到对应 Thought
   - 显示关联强度和方式
```
